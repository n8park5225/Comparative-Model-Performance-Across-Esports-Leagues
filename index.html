Step 3: Assessment of Missingness

Step 3A: MNAR Analysis

I believe visionscore may be MNAR (Missing Not At Random). Vision score depends on warding and other vision actions that are driven by team strategy and in-game performance. If a team is far behind, they may place fewer wards and take fewer vision-related actions, which can lead to a lower or missing vision score. This means the probability that visionscore is missing may depend on the true (unobserved) value of visionscore itself, which is consistent with MNAR.

To better explain this missingness and potentially make it closer to MAR, it would help to have additional data such as team gold difference over time, map control indicators, or a timeline of ward placements. These variables would capture game state and team behavior that could explain why vision activity (and therefore vision score) is missing.

Step 3B: Missingness Dependency

To study whether the missingness of visionscore depends on other variables, I created an indicator column: visionscore_missing, which is True when visionscore is missing and False otherwise. I then compared game outcomes and performance metrics between these two groups.

The plot below compares the distribution of kills when visionscore is missing versus not missing. If the distributions differ noticeably, this suggests that missingness is not completely random and is associated with in-game performance.

Step 5: Framing a Prediction Problem

Prediction Problem

The goal of this project is to predict whether a team will win a professional League of Legends match based on team-level performance metrics recorded during the game.

Response Variable

The response variable is result, which indicates whether a team won or lost:

  • result = 1: Win
  • result = 0: Loss

Problem Type

This is a binary classification problem because the response variable has two possible outcomes (win or loss).

Features Used for Prediction

To predict result, I will use team-level performance metrics such as:

  • kills, deaths, assists
  • goldearned (team economy)
  • towers, inhibitors (structure objectives)
  • dragons, barons (neutral objectives)
  • visionscore (vision control)

Metric for Evaluating the Model

I will evaluate model performance using accuracy and F1-score. Accuracy is easy to interpret, while F1-score can be more informative if the classes are imbalanced because it accounts for both precision and recall.

Time of Prediction

The features listed above are performance statistics that are available by the end of a match. This project predicts match outcomes using these measurable in-game metrics, without using the result itself as an input feature.

Step 8: Fairness Analysis

Fairness Question

Does my final model perform worse for teams in one league compared to teams in other leagues? To make this concrete, I compared model performance for teams in the LCK versus teams in all other leagues.

Groups

  • Group X: LCK teams
  • Group Y: Non-LCK teams (all other leagues)

Evaluation Metric

This is a binary classification task (predicting result: win/loss), so I used the F1-score as my evaluation metric. F1-score is useful because it balances precision and recall, and it is a good choice when class imbalance may be present.

Null and Alternative Hypotheses

Null Hypothesis (H0): The model is fair across these groups; the difference in F1-score between LCK and non-LCK is approximately 0, and any observed difference is due to random chance.

Alternative Hypothesis (H1): The model is not fair across these groups; the F1-score differs between LCK and non-LCK.

Test Statistic

My test statistic is the difference in F1-score between the two groups:

Test statistic: F1(LCK) − F1(non-LCK)

Permutation Test Setup

To test fairness, I performed a permutation test. I kept the model predictions fixed, and randomly shuffled the league labels to simulate a world where league membership does not affect model performance. For each shuffle, I recomputed the F1-score difference between the two groups and compared it to the observed difference.

Results

  • F1(LCK): LCK_F1
  • F1(non-LCK): NON_LCK_F1
  • Observed difference: OBS_DIFF
  • Permutation test p-value: P_VALUE

Conclusion

Using a significance level of 0.05, the p-value is greater than 0.05, so I fail to reject the null hypothesis. The observed F1-score difference between LCK and non-LCK is small and is plausibly explained by random chance. Based on this test and metric, I do not have strong evidence that my model performs worse for one of these groups.
